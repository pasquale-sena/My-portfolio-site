The ADDIE Process

From analysis to deployment.

01. Analysis (The Why)

Traditional onboarding for high-volume agents relies on static modules which fail to simulate real-time pressure or assess baseline skills.

  • Target Audience: New Hire Customer Service Agents (0-2 years exp).
  • Problem: Linear modules cannot accurately measure empathy or de-escalation tactics.
  • Solution: A micro-learning module (< 15 mins) that runs in a standard browser.

02. Design (The Blueprint)

I developed an **"AI-OS"** concept where the learner is not just taking a course, but "booting up" their new workstation.

  • Adaptive Pathing: Novices route to empathy drills; experienced pros skip to complex scenarios.
  • Aesthetic: "Iron Man's HUD meets Corporate Training" using Glassmorphism and terminal effects.
  • Model: L.E.A.R.N. (Listen, Empathize, Apologize, React, Notify) for de-escalation.

03. Development (The Build)

Moved beyond authoring tools to a custom **React Application** for zero-latency interactions and rapid, responsive UI development (Tailwind CSS).

  • Stack: React, Tailwind CSS, Custom State Machine.
  • Scenario: "The Missing Sea View" - High value, high stress context.
  • Logic: Branching paths track "Empathy Points" vs "Efficiency Points".

04. Implementation (Delivery)

The simulation is hosted via GitHub Pages and designed for seamless integration into any host LMS.

  • Hosting: Deployed via GitHub Pages for instant access.
  • Accessibility: Full keyboard navigation and high-contrast text ratios (WCAG AA).
  • Integration: Conceptually wrapped for SCORM 1.2 to report scores.

05. Evaluation (Metrics)

Success is measured by performance metrics that directly translate to job performance.

  • KPIs: Completion Rate, Empathy Index, and Time-to-Resolution (Efficiency).
  • Feedback Loop: If users fail a specific question, the preceding hint text is automatically rewritten for clarity.
