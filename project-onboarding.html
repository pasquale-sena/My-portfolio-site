The ADDIE Process

From analysis to deployment.

01. Analysis (The Why)

[cite_start]Traditional onboarding often relies on static PDFs, which fail to simulate the pressure of live support[cite: 7].

    [cite_start]
  • Target Audience: New Hire Agents (0-2 years exp)[cite: 13].
    • [cite_start]
  • Problem: Linear modules cannot accurately measure empathy or de-escalation tactics[cite: 10].
    • [cite_start]
  • Solution: A micro-learning module (< 15 mins) that runs in a standard browser[cite: 17, 18].

02. Design (The Blueprint)

[cite_start]I developed an "AI-OS" concept where the learner isn't just taking a course, but "booting up" their new workstation[cite: 26, 28].

  • Adaptive Pathing: Novices route to drills; [cite_start]Pros skip to complex scenarios[cite: 30, 31].
    • [cite_start]
  • Aesthetic: "Iron Man's HUD meets Corporate Training" using Glassmorphism and terminal effects[cite: 38].
  • Model: L.E.A.R.N. (Listen, Empathize, Apologize, React, Notify) [cite_start][cite: 25].

03. Development (The Build)

[cite_start]Moved beyond authoring tools to a custom React Application for zero-latency interactions[cite: 41].

    [cite_start]
  • Stack: React, Tailwind CSS, Custom State Logic[cite: 4, 42].
    • [cite_start]
  • Scenario: "The Missing Sea View" - High value, high stress context[cite: 45, 46].
    • [cite_start]
  • Logic: Branching paths track "Empathy Points" vs "Efficiency Points"[cite: 43].

04. Implementation (Delivery)

Designed for accessibility and seamless integration.

    [cite_start]
  • Hosting: Deployed via GitHub Pages for instant access[cite: 56].
    • [cite_start]
  • Accessibility: Full keyboard navigation and high-contrast text ratios (WCAG AA)[cite: 58].
    • [cite_start]
  • Integration: Conceptually wrapped for SCORM 1.2 to report scores to LMS[cite: 57].

05. Evaluation (Metrics)

Success is measured by performance, not just completion.

    [cite_start]
  • Empathy Index: Average score on soft-skill specific choices[cite: 65].
    • [cite_start]
  • Completion Rate: Percentage of users resolving the ticket without "hanging up"[cite: 64].
    • [cite_start]
  • Feedback Loop: If 40% fail a specific node, the preceding hint text is rewritten[cite: 69].
