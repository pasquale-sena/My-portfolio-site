Key Findings

This project fundamentally shifts the approach to Cybersecurity Defense-in-Depth (DiD) education by creating a high-fidelity, interactive Tower Defense (TD) simulation. The solution moves beyond static diagrams to an environment where the consequences of misaligned security controls are immediately visible and measurable.

The core design implements a Defense-in-Depth Layering Model, demonstrating that no single control (like a simple firewall) can mitigate all modern threats (phishing, zero-day). From an architectural standpoint, the use of **p5.js** integrated with modern JavaScript modules ensures high-performance, geometric rendering and clean separation of concerns. This blending of advanced rendering technology with instructional design showcases an ability to manage the full product lifecycle, from needs analysis to high-performance execution.

The ADDIE Process

From analysis to deployment.

01. ANALYSIS (THE WHY)

Traditional cybersecurity training often relies on passive multiple-choice tests that fail to build strategic decision-making or demonstrate the concept of security layers.

  • TARGET AUDIENCE:
    Security Analysts, IT Administrators, and Developers learning foundational DiD principles.
  • PROBLEM:
    How can a learner visualize the layered failure/success of different security controls against specific attack vectors?
  • SOLUTION:
    A complex, multi-file Tower Defense prototype where Node placement must correspond to the real-world vulnerability of the incoming Threat.

02. DESIGN (THE BLUEPRINT)

I developed a Cyberpunk Data Stream concept where the learning goal is to build a layered defense strategy (Defense-in-Depth) over a visual data path.

  • INSTRUCTIONAL MAPPING:
    Defense Nodes (Firewall, IDS/WAF) map to Security Controls; Threats (Malware, Phishing Bot) map to Attack Vectors.
  • MECHANIC:
    Success requires placing nodes in layered sequence (e.g., Firewall to slow a threat, giving a high-damage IDS time to fire).
  • AESTHETIC:
    "High-Contrast Digital Terminal" featuring Neon Cyan data streams and electric green UI text to create an immersive, high-stakes atmosphere.

03. DEVELOPMENT (THE BUILD)

The prototype was built using a multi-file modular structure to showcase mastery of modern JavaScript classes and principles.

  • STACK:
    p5.js (Canvas) for high-performance game loop; JavaScript (ES6 Modules) for entity classes.
  • CORE LOGIC:
    Threats track Status Effects (e.g., SLOWED); Defense Nodes use complex targeting checks to mimic real-world control limitations.
  • VISUAL FEEDBACK:
    Transient VFX (lasers, explosions) and pulsing fields communicate node activity instantly.

04. IMPLEMENTATION (DELIVERY)

The game is hosted on GitHub Pages and features multiple layers of accessible instruction designed to seamlessly integrate into any learning ecosystem.

  • HOSTING:
    Deployed via GitHub Pages for instant public access and portfolio viewing.
  • LEARNING INTEGRATION:
    Instructional Pause System with dynamic pop-ups explains new threats and optimal strategies.
  • MODULARITY:
    Structure is ready to be compiled/wrapped for SCORM reporting across an enterprise LMS.

05. EVALUATION (METRICS)

Success is measured by the learner’s ability to apply DiD principles strategically, which translates directly to educational effectiveness.

  • KPIs:
    Strategic Alignment Index (SAI), Completion Rate, and Budget Efficiency (rewarding strategic placement).
  • FEEDBACK LOOP:
    Failure to stop a specific threat highlights the need to revisit the corresponding security control rationale.
